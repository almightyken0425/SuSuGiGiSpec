# 產品文件管理

## 產品規格 Product Specs

### Screens

- **目的:** 定義所有使用者可見的 畫面 與 互動。
- **內容規則:**
    - **應包含:** 所有使用者畫面的 UI 佈局、元件結構、互動邏輯、與導航流程。
    - **禁止:** 描述業務計算邏輯、資料庫儲存邏輯。

### Background Logics

- **目的:** 定義所有 背景運作 的業務邏輯, 這些邏輯不直接依賴 UI。
- **內容規則:**
    - **應包含:** 所有背景運作的業務邏輯、流程、與資料轉換。
    - **禁止:** 描述 UI 佈局、資料庫 Schema。

### Models

- **目的:** 定義 App 唯一的資料事實來源 Single Source of Truth。
- **內容規則:**
    - **應包含:** 所有資料庫的 Schema 定義、欄位、型別、索引，靜態 JSON 定義檔的結構。
    - **禁止:** 描述業務邏輯如何 使用 這些資料，UI 互動如何 觸發 這些資料。

## 專案管理 Project Management

### Policy

- **目的:** 定義團隊協作的 規範。
- **內容規則:**
    - 所有與 如何撰寫 和 如何協作 相關的規範。
    - 規格撰寫風格、Git 流程、程式碼風格。

### Planning

- **目的:** 定義 開發 的藍圖與範疇。
- **內容規則:**
    - MVP 範圍、開發階段性藍圖、或檔案結構規劃。
