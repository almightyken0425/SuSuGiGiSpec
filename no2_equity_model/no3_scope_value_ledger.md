# 範疇與價值帳本

## 核心目標

- 本文件定義產品的完整藍圖, 包含產品組件與營運事項
- 依據股權原則, 評估每個範疇的理論價值點數
- 這是團隊賺取貢獻點數的任務總表

## 營運事項：純營運價值

- **說明:**
    - 此處列出產品上線後, 非功能開發的持續性營運活動, 例如公司治理、行銷推廣、法律合規等
    - 估算點數應以年或季為單位, 作為速率
    - 營運事項的點數釋放，僅在產品正式上線後開始計算
- **持續性產品營運**
    - **價值主張:** 維持產品的穩定運作、用戶增長與商業變現，確保公司永續經營
    - **點數算法:** /月
    - **涵蓋範疇:** 參見角色定義中各角色的職責描述
- **技術維護與升級**
    - **價值主張:** 因應作業系統升級、第三方套件更新與安全性修補，確保軟體資產不貶值
    - **點數算法:** /月
    - **涵蓋範疇:** 包含 OS 版本適配、Library 升級、重構與效能優化

## 施工事項：純施工價值

- **說明:**
    - 此處定義所有從 0 到 1 打造產品功能所需的一次性勞力投入
    - 估算點數為一次性總額, 於功能開發完成並上線時發放
    - 用於獎勵早期開發階段的高密度貢獻
- **全產品初始建置**
    - **價值主張:** 將產品願景轉化為具體的軟體資產，建立市場進入門檻與核心競爭力
    - **點數算法:** /次
    - **涵蓋範疇:** 參見角色定義中開發與設計角色的職責描述
