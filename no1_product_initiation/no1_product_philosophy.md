# 產品理念與使用者旅程

## 核心洞察：記帳的演化

- 記帳行為本身是無趣且違反人性的
- 產品的價值主張, 會隨著使用者的心態成熟而轉變
- 產品設計必須應對使用者在不同階段的期望

---

## 使用者旅程 User Journey

- **獲取 Acquisition**
    - **目標:** 降低首次使用門檻
    - **使用者心態:** 記帳是件很無趣的事情
    - **產品策略:**
        - **活潑的介面:** 必須提供愉悅、美觀、活潑的 UI, 讓使用者在首次使用時心情很好, 建立正面第一印象
        - **降低挫折感:** 最大限度地簡化首次記帳流程
- **留存 Retention**
    - **目標:** 建立長期使用習慣
    - **使用者心態:** 我需要一個快速、方便的工具
    - **產品策略:**
        - **速度與便利性:** 記帳的速度與便利性, 是決定使用者能否永久留下來的關鍵
        - **無摩擦體驗:** 記帳應成為使用者的肌肉記憶與反射動作, 而非每天的負擔
- **中期價值 Mid-Term Value**
    - **目標:** 提供初步的財務洞察
    - **使用者心態:** 我想知道錢都花到哪裡去了
    - **產品策略:**
        - **清晰的報表:** 提供直覺的圓餅圖與列表
        - **回答問題:** 協助使用者回答 哪些地方花得多, 應該從哪裡節省
- **覺察 Realization**
    - **目標:** 應對使用者的挫折感
    - **使用者心態:**
        - 知道自己花得多, 根本沒有用
        - 該花的生活費還是會花, 記帳無法改變支出
    - **產品策略:**
        - **接受現實:** 產品不應強迫使用者進行痛苦的節約
        - **轉向:** 提供更深層的價值, 例如資產管理與現金流預測
- **核心價值 Core Value**
    - **目標:** 提供即時的決策依據與償債能力評估
    - **使用者心態:**
        - 我想買車或進行大額消費, 我現在的財務狀況允許嗎
        - 既然支出無法輕易改變, 那我真正的核心問題是這個月到底還剩下多少錢可以運用
    - **產品策略:**
        - **強調結餘:** App 的所有報表與計算, 最終都應導向「結餘」
        - **決策輔助:** 幫助使用者在消費當下判斷該筆支出是否會影響短期現金流
- **預見與連結 Foresight & Connection**
    - **目標:** 提供中長期的戰略規劃與市場定位
    - **概念:** 個人即公司 Personal Enterprise, 記帳是為了經營人生這家公司
    - **使用者心態:**
        - 如果我未來 3 到 5 年要買房或結婚, 我現在的開銷跟收入應該要做什麼結構性調整
        - 我想知道我的財務健康度與財富累積速度, 在同齡人或社會中處於什麼位置
    - **產品策略:**
        - **戰略規劃:** 提供 3 至 5 年的財務模擬功能, 讓使用者進行情境分析, 例如 買房 vs 租房 對未來現金流的影響
        - **市場對標:** 引入類似企業 EPS 的個人財務指標, 允許使用者在匿名且安全的前提下, 與大數據中的相似群體進行比較, 確認自身的階級流動狀況
- **駕馭複雜性 Mastering Complexity**
    - **目標:** 解決真實人生中無法被簡單 UI 涵蓋的複雜財務邏輯
    - **背景:**
        - 使用者的財務狀況會隨年齡增長而指數級複雜化, 包含 貸款、股票投資、多樣化被動收入
        - 家庭結構改變導致的複雜協作, 例如 夫妻共用錢包、依收入比例 7:3 分攤開銷、代墊款項結算
    - **使用者心態:** 手機 App 的簡單篩選器已經無法滿足我客製化的報表需求
    - **產品策略:**
        - **Web 進階視圖:** 提供大螢幕、高密度的資料管理介面
        - **可程式化財務:** 引入類似 JIRA JQL 的查詢語言, 允許使用者定義極度複雜的邏輯, 例如 篩選所有標記為家庭支出的項目並自動計算 30% 歸屬額, 讓進階使用者能自由定義專屬的財務報表

## 數據變現與總體經濟 Data Monetization & Macro Economy

- **雙邊市場策略 Two-Sided Market Strategy**
    - **定義:** 將產品從單純的 C 端工具, 轉型為連接 個人微觀行為 與 市場宏觀趨勢 的數據橋樑
    - **原則:**
        - **隱私絕對優先:** 所有 B 端應用僅能存取 經過去識別化 De-identified 與 聚合 Aggregated 的統計數據, 嚴禁洩漏任何個人 PII 資訊
        - **互惠循環:** B 端的營收將回饋於 C 端產品的研發, 讓一般使用者能以更低成本享受高品質服務
- **市場情資儀表板 Market Intelligence Console**
    - **目標受眾:** 市場研究機構、品牌商、金融投資機構
    - **價值主張:** 提供真實、即時的消費者行為趨勢與總體資產流動圖譜
    - **產品型態:**
        - **複雜 Web UI:** 提供強大的資料篩選、交叉分析與視覺化工具
        - **查詢維度:** 允許 B 端客戶查詢特定群體 如 25-30歲都會男性 的消費趨勢、資產配置比例等聚合數據
- **總經數據串流 Macro Data API**
    - **目標受眾:** 銀行風控系統、量化交易團隊、學術研究單位
    - **價值主張:** 將聚合後的記帳數據視為一種 另類數據 Alternative Data, 用於修正傳統經濟指標的滯後性
    - **產品型態:**
        - **API Service:** 提供標準化的 RESTful API 或 WebSocket 串流
        - **數據輸出:** 輸出特定產業的消費指數、不同地區的通膨實況等總體經濟指標
