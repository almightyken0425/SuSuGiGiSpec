# 營運操作標準流程 (Operation SOP)

## 1. 序號發放流程

### 1.1 需求申請
- **申請人**: 行銷部門、業務部門。
- **申請方式**: 填寫「序號申請單」(Google Form 或內部工單)。
- **必要資訊**:
  - 用途 (e.g., 合作夥伴贈送)
  - 數量
  - 權益類型 (Tier / 天數)
  - 預計發放日期

### 1.2 核決權限
- **一般申請**: 營運經理核准。
- **大量申請 (>100 組)**: 需產品負責人 (PO) 核准。

### 1.3 生成執行
- **執行者**: 指定的營運人員 (Admin)。
- **工具**: 依照 `no1_firebase_console_guide.md` 操作 Firebase Console。
- **產出**: 將生成的序號整理至 Excel/CSV 回傳給申請人。

### 1.4 發放記錄
- 執行者需在「序號發放登記表」記錄：
  - 申請單號
  - 生成時間
  - 序號範圍/清單連結
  - 經手人

## 2. 異常處理流程

### 2.1 用戶回報序號無效
1. **客服接收**: 收集用戶提供的序號與錯誤訊息截圖。
2. **查詢狀態**: 營運人員至 Firebase Console 查詢該序號。
   - 若 `isActive` 為 false: 確認是否被停用。
   - 若 `currentRedemptions` 已滿: 確認是否被盜用。
   - 若查無此號: 確認用戶是否輸入錯誤 (O/0, I/1)。
3. **回覆用戶**:
   - 若為輸入錯誤 -> 引導正確輸入。
   - 若為系統問題 -> 補發新序號，並將舊序號停用。

### 2.2 序號外洩緊急處理
1. **通報**: 發現序號被惡意大量兌換。
2. **緊急停用**:
   - 立即將該批次序號的 `isActive` 設為 `false`。
3. **損害控管**:
   - 撈取已兌換該批序號的 User ID。
   - 評估是否回溯權益 (需法務確認)。
