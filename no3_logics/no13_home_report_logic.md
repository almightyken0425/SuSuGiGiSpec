# 首頁報表邏輯: HomeReportLogic

## 目的

- 定義首頁報表的資料聚合、色票分配與分組規則

---

## assignChartColors 分配圖表色票

- **輸入:**
  - 圖表來源對應的各分組金額清單
- **執行:**
  - 依金額由大到小排序
  - 計算截止門檻：所有分組金額總和的 5/6
  - 從最大分組開始依序累加，以下條件先達成者為截止點：
    - 累積金額達到門檻
    - 已累加至第 5 個分組
  - 截止點以內的分組依排名分別分配 `chart[0]` 至 `chart[N-1]`，N 為截止分組數
  - 截止點以外的分組合併為其他，分配 `primary[300]`
- **回傳:**
  - 各分組識別碼對應的色票清單
  - 供環形圖繪製的各區塊名稱、金額與色票清單

---

## buildPeriodReport 產生期間報表

- 查詢指定期間的交易與轉帳，依篩選條件聚合後回傳分組資料與圖表資料
- **輸入:**
  - 期間起訖日期
  - 已選帳戶清單
  - 圖表來源
  - 列表分組模式
- **性質:**
  - 非同步，需讀取本地資料庫
- **轉帳分類:**
  - **條件:**
    - 逐筆轉帳依已選帳戶清單判斷顯示方式
  - **執行:**
    - **IF** 轉出與轉入帳戶皆在已選清單:
      - 略過，不列入計算
    - **IF** 轉出與轉入帳戶皆不在已選清單:
      - 略過，不列入計算
    - **IF** 外部帳戶已刪除或已停用:
      - 略過，不列入計算
    - **IF** 轉出帳戶在已選清單:
      - 以 amountFrom 列入支出
    - **ELSE:**
      - 以 amountTo 列入收入
- **幣別模式偵測:**
  - **IF** 所有已選帳戶使用相同 currencyCode:
    - 以單幣別模式計算，金額使用原始幣別
  - **ELSE:**
    - 以多幣別模式計算，金額換算為 baseCurrencyId 對應幣別後累計
- **類別分組色票指派:**
  - **執行:**
    - 呼叫 assignChartColors，傳入與圖表來源一致的類型金額清單，取得 primary 類型各分組的色票
    - primary 類型超出截止點的分組：分配 `primary[300]`
    - secondary 類型，即與圖表來源相反的所有分組：分配 `primary[800]`
- **日期分組色票指派:**
  - **執行:**
    - primary 類型：依日期由新到舊的排序位置分配 `chart[0]` 至 `chart[N-1]`，超出 chart 色票數量的部分分配 `primary[300]`
    - secondary 類型：全部分配 `primary[800]`
- **支出與收入區排列:**
  - **IF** 圖表來源為支出:
    - 支出區排在收入區之前
  - **ELSE:**
    - 收入區排在支出區之前
- **回傳:**
  - 各分組資料清單，含分組色票、分組標題、各筆紀錄
  - 環形圖資料
  - 期間餘額，計算方式為收入合計減支出合計
  - 幣別模式
