# 產品理念與使用者旅程

## 核心洞察：記帳的演化

- 記帳行為本身是無趣且違反人性的
- 產品的價值主張, 會隨著使用者的心態成熟而轉變
- 產品設計必須應對使用者在不同階段的期望

## 階段一：獲取 (Acquisition)

- **目標:** 降低首次使用門檻
- **使用者心態:**
    - 記帳是件很無趣的事情
- **產品策略:**
    - **活潑的介面:**
        - **行為:** 必須提供愉悅、美觀、活潑的 UI
        - **目的:** 讓使用者在首次使用時心情很好, 建立正面第一印象
    - **降低挫折感:**
        - **行為:** 最大限度地簡化首次記帳流程

## 階段二：留存 (Retention)

- **目標:** 建立長期使用習慣
- **使用者心態:**
    - 我需要一個快速、方便的工具
- **產品策略:**
    - **速度與便利性:**
        - **行為:** 記帳的速度與便利性, 是決定使用者能否永久留下來的關鍵
        - **目的:** 讓記帳成為一個反射動作, 而非負擔

## 階段三：中期價值 (Mid-Term Value)

- **目標:** 提供初步的財務洞察
- **使用者心態:**
    - 我想知道錢都花到哪裡去了
- **產品策略:**
    - **清晰的報表:**
        - **行為:** 提供直覺的圓餅圖與列表
        - **目的:** 回答使用者 哪些地方花得多, 應該從哪裡節省

## 階段四：覺察 (Realization)

- **目標:** 應對使用者的挫折感
- **使用者心態:**
    - 知道自己花得多, 根本沒有用
    - 該花的生活費還是會花, 記帳無法改變支出
- **產品策略:**
    - **接受現實:**
        - **行為:** 產品不應強迫使用者進行痛苦的節約
        - **目的:** 轉而提供更深層的價值

## 階段五：核心價值 (Core Value)

- **目標:** 提供最終的財務答案
- **使用者心態:**
    - 既然支出無法輕易改變, 那我真正的核心問題是...
    - 我這個月到底還剩下多少錢可以運用
- **產品策略:**
    - **強調結餘:**
        - **行為:** App 的所有報表與計算, 最終都應導向「結餘」
        - **目的:** 幫助使用者從「控制支出」轉向「管理現金流」
