# 畫面規格書專屬規範

## 核心原則

- **隱含順序:** 條列的視覺順序即代表其在 UI 上的佈局順序，由上至下，由左至右
    - 條列順序同時決定元件的渲染位置，無需額外標注方位詞

---

## 標準章節結構

- **必要章節:**
    - `## 畫面目標` 簡述畫面的核心功能
    - `## UI 佈局` 描述畫面的視覺結構與元件組成
    - `## 核心邏輯` 描述資料處理、互動行為、業務邏輯
    - `## 導航` 定義進入與退出的導航行為
- **可選章節:**
    - `## 權限邏輯` 若有特定權限檢查邏輯
    - `## 錯誤處理` 若有特殊錯誤處理邏輯

---

## 元件命名規範

- **格式:** 元件類型加功能描述
- **規則:** 所有 UI 元件名稱必須使用反引號包裹
- **範例:**
    - `取消按鈕`
    - `轉出帳戶選擇器`
    - `時間選擇器`
    - `金額輸入框`
- **禁止:** 使用技術術語作為元件唯一描述

---

## 屬性標記

- **必填與可選:**
    - 使用粗體標記元件的必填或可選狀態
    - 未標記必填的元件均預設為可選
    - **範例:** `**屬性:** 必填` 或 `**屬性:** 可選`
- **其他屬性類型:**
    - **樣式** 描述視覺樣式
    - **鍵盤** 描述鍵盤類型
    - **可見性** 描述顯示條件
    - **權限** 描述權限要求

---

## 導航段落

- **進入:**
    - **來源** 列出可能的來源畫面
    - **必要參數** 列出必須傳遞的參數
    - **可選參數** 列出可選的參數
- **退出:**
    - **觸發** 描述退出的觸發條件
    - **行為** 描述退出的導航行為
    - **回傳** 若有回傳值，描述回傳內容
