# 股權分配原則

## 核心目標

- 本文件定義團隊以勞力換取股份的價值計算基礎
- 建立一個客觀標準, 用於衡量所有成員的貢獻
- 確保股權分配公平、透明、且具動態性

## 估價原則

- **總理論價值**
    - 服務的總理論價值, 等於若將完整藍圖外包給接案公司所需的總費用
- **組件估價**
    - 總理論價值 ＝ 產品組件價值 ＋ 營運角色價值
    - 總理論價值將被換算為 100% 的總貢獻點數

## 動態股權分配

- **股權計算**
    - 任何成員的股權百分比 ＝ 該成員的總貢獻點數 / 團隊的總貢獻點數
- **貢獻點數**
    - 貢獻點數的獲取方式, 定義於 `no2_scope_value_ledger.md` 範疇與價值
    - 本文件旨在定義所有可用於賺取貢獻點數的任務範疇

## 核心情境處理

- **情境：平衡一次性建置 vs 持續性營運**
    - **定義:** 
      - 產品組件價值如 App 開發是一次性的總額貢獻點數
      - 營運角色價值如行銷是持續性的速率貢獻點數, 例如每年 X 點
    - **帳本:** `no2_scope_value_ledger.md` 必須明確定義營運角色的價值計算週期, 例如年
    - **平衡:**
        - 建置期, 開發角色的點數獲取速度快
        - 營運期, 開發角色的點數總額被賺取完畢, 轉由營運角色持續獲取點數
        - 兩者皆透過貢獻點數在同一個基準上進行計算
- **情境：處理範疇擴增**
    - **定義:** 產品的總理論價值並非固定, 而是隨著團隊決議而演進
    - **流程:**
        - 團隊共同決議擴增範疇, 例如開發新功能
        - 團隊必須共同估算新範疇的理論價值點數
        - 必須將新範疇與其價值點數, 新增至 `no2_scope_value_ledger.md`
        - 服務的總理論價值分母因此增加
        - 負責執行新範疇的成員, 依據其貢獻賺取新釋出的點數

## 貢獻類型

- **時間貢獻**
    - 團隊成員投入的實際工作時數
    - 依據其核心角色的理論時薪換算為貢獻點數
- **現金貢獻**
    - 團隊成員代墊的營運費用
    - 範例: 伺服器費用、開發者帳號年費、購買的軟體授權
    - 現金貢獻應以加權倍數換算為貢獻點數, 因為現金在早期是最高風險的投入

## 股份分配應用

- **多重角色貢獻**
    - 成員 A 可同時擔任 `前端開發` 與 `行銷與成長` 角色
    - 該成員的總貢獻點數, 為其在所有角色投入貢獻的加總
- **範例情境**
    - 團隊估算 Web 平台總價值為 50 萬理論薪資
    - **IF** 團隊決議外包, 支付 50 萬現金
        - 該 50 萬計入代墊成員的現金貢獻
    - **IF** 成員 B 決議自行開發
        - 成員 B 在開發 Web 平台的過程中, 可依其理論時薪賺取貢獻點數, 直至 50 萬的價值上限
