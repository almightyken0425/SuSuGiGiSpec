# 輸入欄位盤點清單 Input Field Inventory

本文件盤點 Accounting App 中所有使用者輸入欄位，並對照 `input_field_limits_strategy.md` 檢視現狀與缺口。

## 1. 文字輸入欄位 (Text Inputs)

此表釐清「畫面規格」與「策略文件」之間的差異。

| 欄位名稱 | 畫面規格現狀 (Screen Spec) | 策略文件現狀 (Strategy Doc) | 差異分析 (Gap Analysis) |
| :--- | :--- | :--- | :--- |
| **帳戶名稱** | ✅ **有** (AccountEditor) | ✅ **有** (上限 255) | **一致** |
| **類別名稱** | ✅ **有** (CategoryEditor) | ✅ **有** (上限 255) | **一致** |
| **交易備註** | ✅ **有** (TransactionEditor) | ✅ **有** (上限 1000) | **一致** |
| **轉帳備註** | ✅ **有** (TransferEditor) | ✅ **有** (上限 1000) | **一致** |
| **搜尋關鍵字** | ✅ **有** (SearchScreen) | ❌ **無** | **策略缺漏**: 畫面有此功能，但策略文件漏掉定義限制。 |
| **帳戶描述** | ❌ **無** (AccountEditor) | ✅ **有** (上限 500) | **規格缺漏?**: 策略預期有此欄位，但目前畫面設計沒做。 |
| **類別說明** | ❌ **無** (CategoryEditor) | ✅ **有** (上限 500) | **規格缺漏?**: 策略預期有此欄位，但目前畫面設計沒做。 |
| **週期交易名稱** | ❌ **無** (RecurringSetting) | ✅ **有** (上限 255) | **規格缺漏?**: 策略預期有此欄位，但目前畫面設計沒做。 |
| **標籤名稱** | ❌ **無** (無相關畫面) | ✅ **有** (上限 100) | **功能缺漏**: 目前 App 規格完全無 Tag 功能。 |

## 2. 特殊格式輸入 (Special Inputs)

| 欄位名稱 | 畫面規格現狀 | 驗證規則 |
| :--- | :--- | :--- |
| **兌換序號** | ✅ **有** (RedeemCodeScreen) | 限制 14 chars (含分隔線), 僅限英數大寫 |
| **電子郵件** | ✅ **有** (LoginScreen) | 依賴 Google SDK 驗證 |

## 3. 數值輸入欄位 (Numeric Inputs)

需評估最大數值限制 (Max Value) 與小數位數 (Decimal Places)，而非字元長度。

| 欄位名稱 | 所屬畫面 | 類型 | 備註 |
| :--- | :--- | :--- | :--- |
| **初始餘額** | AccountEditor | 金額 | 需定義最大金額 (e.g., 999,999,999) |
| **交易金額** | TransactionEditor | 金額 | 同上 |
| **轉出金額** | TransferEditor | 金額 | 同上 |
| **轉入金額** | TransferEditor | 金額 | 同上 |
| **匯率** | Account/Transfer Editor | 匯率 | 需定義小數位數 (e.g., 4位) |
| **週期頻率** | RecurringSetting | 數字 (間隔) | 最小 1, 需定義最大值 (e.g., 999) |

## 4. 建議行動

1.  **確認多餘的策略項目**: 請確認「帳戶描述」、「類別說明」、「週期交易名稱」是否為未來功能？
    *   若**是**: 需修改畫面規格 (Spec)，補上這些輸入框。
    *   若**否**: 需修改策略文件 (Strategy)，移除這些多餘的定義。
2.  **補齊策略缺漏**: 請在策略文件中加入「搜尋關鍵字」的字數限制 (建議 100 字)。
3.  **確認 Tag 功能**: 目前規格無 Tag 功能，建議從策略文件中暫時移除「標籤名稱」。
