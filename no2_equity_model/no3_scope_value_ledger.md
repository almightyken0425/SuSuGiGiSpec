# 範疇與價值帳本

## 核心目標

- 本文件定義產品的完整藍圖, 包含產品組件與營運事項
- 依據 `no1_equity_principles.md` 原則, 評估每個範疇的理論價值點數
- 這是團隊賺取貢獻點數的任務總表

## 營運事項：純營運價值

- **說明:**
    - 此處列出產品上線後, 非功能開發的持續性營運活動, 例如公司治理、行銷推廣、法律合規等
    - 估算點數應以年或季為單位, 作為速率
    - 營運事項的點數釋放，僅在產品正式上線後開始計算
- **持續性產品營運**
    - **價值主張:** 維持產品的穩定運作、用戶增長與商業變現，確保公司永續經營
    - **估算點數:** TBD / 每年
    - **涵蓋範疇:** 參見 `no3_role_definitions.md` 中各角色的職責描述

## 施工事項：純施工價值

- **說明:**
    - 此處定義所有從 0 到 1 打造產品功能所需的一次性勞力投入
    - 估算點數為一次性總額, 於功能開發完成並上線時發放
    - 用於獎勵早期開發階段的高密度貢獻
- **全產品初始建置**
    - **價值主張:** 將產品願景轉化為具體的軟體資產，建立市場進入門檻與核心競爭力
    - **估算點數:** TBD / 一次性
    - **涵蓋範疇:** 參見 `no3_role_definitions.md` 中開發與設計角色的職責描述

## 產品組件：雙重價值模型

- **說明:**
    - 本區塊詳細列出每個功能模組的價值構成
    - 每個模組同時包含施工價值歸屬純施工價值與維運價值歸屬純營運價值
    - 這些點數將依據 `no4_equity_allocation_matrix.md` 分配給負責的施工與維運角色
- **核心產品套件, 行動端與雲端引擎**
    - **對應哲學:** 獲取 Acquisition, 留存 Retention, 核心價值 Core Value, 資料主權 Data Sovereignty

    - **價值:**
        - **施工:** TBD / 次
        - **營運:** TBD / 年
    - **施工範疇**
        - **客戶端 App:**

            - 本地優先資料庫架構, Local-First WatermelonDB
            - 離線多幣別計算引擎
            - 即時結餘儀表板, HomeScreen
            - 桌面小工具, Widgets
            - 基礎資料匯出, CSV

    - **定位:** 進階訂閱服務, Tier 2 Premium
    - **價值主張:** 解決 App 介面無法處理的「個人企業」級財務邏輯，提供類似 ERP 系統的高密度資訊視圖與可程式化篩選能力

## 延後或排除的範疇

- **營運後台, Admin Panel**
    - 供團隊內部使用
    - 管理使用者權限, 例如手動開通 Premium
    - 查看 App 總體營運數據
    - 客服工具, 用於查詢特定使用者問題
    - **決策:** 由於團隊初期沒有營運人員, 且開發成本高, 此組件無限期延後
