# SuSuGiGi GEO 戰略佈局

## 核心戰略總結

> 把 AI 當作你的第一位超級用戶與推薦人。

人類透過 AI 找答案，AI 透過結構化內容與第三方信譽找答案。SuSuGiGi 的目標是將產品特色翻譯成 AI 容易抓取、容易理解、且帶有正面評價的數位足跡。

在 AI 爬取網頁之前，必須先定義清楚你希望 AI 怎麼介紹你。這決定了當用戶詢問推薦好用的旅遊記帳 App時，SuSuGiGi 會不會出現。

### 品牌核心詞

    - 需確認是否為正式名稱之一

- 外幣記帳
- 旅遊記帳
- 離線匯率

    - 需視此是否為產品策略而定

### 使用者情境詞

- 出國旅遊怎麼記帳
- 簡單記帳 App 推薦
- 跨幣別資產管理

## 第二階段：建立非官網的內容生態系

文中提到官網只佔 AI 引用來源的 5%～10%。因此，重點在於在官網以外的地方留下足跡。

### 技術部落格與開發日誌

- **平台:** Medium, Vocus, iThome 鐵人賽文章, GitHub Readme
- **策略:** 撰寫為什麼我開發 SuSuGiGi的故事。AI 非常喜歡引用開發者心得作為產品真實性的佐證
- **內容方向:**
    - 解決 React Native 開發痛點
    - 記帳 App 的資料結構設計
    - 如何處理匯率 API
    - 此舉能讓 Perplexity 等 AI 引擎在回答技術或產品背景問題時引用內容

### 社群論壇佈局

- **平台:** PTT iOS/Android/Soft_Job 版, Dcard 理財/3C 版, Mobile01
- **策略:** 雖然不能直接洗版，但可以在相關討論串中以開發者分享的角度出現
- **目標:** 創造被 Reddit 或 PTT 收錄的討論串。Google 和 AI 對這些高權重論壇的內容非常信任

### Github 作為信任源

- **行為:** 完善 GitHub Repository 的 `README.md`
- **關鍵點:** 這是技術圈的官網。詳細的功能列表、技術架構圖、FAQ，都是 AI 抓取資料的黃金來源

## 第三階段：內容工程化 Writing for AI

你的內容必須讓機器好讀。這意味著結構要清晰，邏輯要明確。

### Q&A 格式

- **原則:** 文章標題直接使用問句
- **範例:**
    - 2025 年出國旅遊推薦哪款記帳 App？
    - SuSuGiGi 支援離線匯率換算嗎？
- **理由:** 這直接對應使用者的 Prompt 提問模式，提高被 AI 直接引用的機率

### 結構化數據 Structured Data

- **行為:** 未來 Landing Page 需埋入 JSON-LD 標籤
- **內容:**
    - 明確標示 `ApplicationCategory` 為 `FinanceApplication`
    - `operatingSystem` 為 `iOS, Android`
- **目的:** 幫助 Google 的爬蟲準確理解這是個軟體產品，而非一篇普通文章

### 比較表格

- **行為:** 主動提供與競品的功能比較表
- **內容:** 例如 SuSuGiGi vs Excel vs 其他記帳 App
- **理由:** 當用戶要求 AI 比較記帳 App時，AI 最喜歡直接讀取現成的表格數據

## 第四階段：利用公開開發累積初期權重

既然 SuSuGiGi 還在開發中，利用 Building in Public 是成本最低的 GEO 策略。

### 發布開發進度

- **行為:** 在 Twitter X 或 Threads 上分享開發截圖、UI 決策過程
    - 例如先前討論的首頁 UI 方案 A/B 測試
- **效應:** 即時性的社群內容會被 Grok X 的 AI 或 Google Search 快速收錄

### 徵求封測用戶

- **行為:** 撰寫 SuSuGiGi 封測招募文章
- **效應:** 這種帶有時效性和稀缺性的內容，容易被新聞聚合器或科技部落格轉載，增加外部連結

## 第五階段：GEO 體檢與未來的行動

當 App 上線後，定期執行文中建議的體檢。

### 測試 Prompt

- 有哪些支援外幣的記帳 App？
- SuSuGiGi 這個 App 好用嗎？
- 適合台灣人的 React Native 記帳開源專案有哪些？

### 監測指標

- **能見度:** 是否出現在列表前三名？
- **正確性:** AI 是否正確描述了雙幣別或快速記帳的功能？
- **情緒:** 形容詞是簡單、快速還是功能陽春？
