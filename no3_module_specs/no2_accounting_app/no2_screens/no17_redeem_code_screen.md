# 序號兌換規格: RedeemCodeScreen

## 畫面目標

- **提供:** 使用者輸入序號介面
- **提供:** 兌換權益 (會員升級、展期等)

## UI 佈局

- **頂部導航列:**
    - **內部元件:**
        - **返回按鈕**
            - **導航:** PreferenceScreen
        - **標題**
            - **內容:** 兌換序號
- **輸入區域:**
    - **序號輸入框:**
        - **UI:** TextField
        - **Placeholder:** 請輸入 12 位數序號
        - **鍵盤:** ASCII Capable / Uppercase
        - **格式:** 自動格式化為 `XXXX-XXXX-XXXX` (每 4 碼加 Dash)
    - **輔助說明:**
        - **內容:** 序號位於您的實體卡片背面或活動信件中
- **操作按鈕:**
    - **立即兌換按鈕:**
        - **狀態 - Disabled:**
            - **條件:** 輸入長度不足 12 碼
        - **狀態 - Loading:**
            - **條件:** API 請求中
- **歷史記錄 (Optional):**
    - **標題:** 最近兌換記錄
    - **列表:**
        - **內容:** 顯示最近 5 筆兌換記錄 (日期 + 項目)

## 核心邏輯

- **輸入驗證:**
    - **即時格式化:**
        - **行為:** 輸入時自動轉大寫
        - **行為:** 自動插入分隔線 (-)
    - **長度檢查:**
        - **限制:** 最大長度 14 字元 (12 碼 + 2 分隔線)
- **兌換流程:**
    - **觸發:** 點擊 立即兌換 按鈕
    - **行為:** 顯示 Loading Spinner
    - **行為:** 呼叫 `POST /api/v1/redeem`
    - **IF** 成功 (Success):
        - **行為:** 隱藏 Loading
        - **行為:** 顯示 成功對話框 (Dialog / BottomSheet)
            - **標題:** 兌換成功！
            - **內容:** 您已升級為 Tier 1 會員，效期至 2024/12/31
            - **動作:** 點擊 確定 返回上一頁或重整 User Profile
    - **IF** 失敗 (Error):
        - **行為:** 隱藏 Loading
        - **行為:** 顯示 錯誤訊息 (Toast 或 Inline Error)
            - **內容:** 序號無效或已過期 (CODE_EXPIRED)

## 狀態管理

- **本地狀態:**
    - `redeemCode`
    - `isLoading`
    - `error`

## 導航

- **進入:**
    - **來源:** PreferenceScreen
- **退出:**
    - **來源:** 頂部導航列 返回按鈕
    - **導航:** PreferenceScreen
- **成功後:**
    - **行為:** 返回上一頁
    - **行為:** 觸發 App 狀態更新 (Refetch User Profile)
