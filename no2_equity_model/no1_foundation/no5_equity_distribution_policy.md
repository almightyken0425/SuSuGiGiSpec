# 股權分配總表

## 核心目標

- 本文件是股權計算的最終總結
- 目的在於將產品組件的任務分配給角色定義的角色
- 藉此計算出每個角色在總理論價值中所佔的價值點數比例
- 這個比例是未來薪資分配與股權結構的依據

---

## 產品利潤中心分配機制

- **核心概念:**
    - 為了讓貢獻者能直接從其負責的產品成功中獲益，我們採用產品利潤中心制度
    - 每個產品組件視為獨立的利潤中心，其產生的營收 Revenue 扣除直接成本後，將優先分配給該組件的價值點數持有者
- **分配邏輯:**
    1. **歸屬認定:** 營收需明確歸屬至特定產品組件，例如 Web Console 訂閱費、AI Advisor 加值服務費
    2. **分潤池:** 該產品的淨利潤形成該產品的分潤池 Dividend Pool
    3. **分配權重:** 個人在該產品的分潤權重 = (個人在該產品持有的價值點數) / (該產品總價值點數)
    4. **範例:** 若 Web Console 產生 $100 淨利，而某 Frontend RD 持有 Web Console 10% 的點數，則他可分得 $10
- **適用範圍:**
    - 此機制適用於 Web 複雜報表與 AI 個人化建議等具備獨立變現能力的加值產品
    - 基礎產品記帳 App 若為免費或低價引流，則其價值主要體現在推動整體生態系成長，可透過公司整體股權分紅彌補

## 現金出資轉換機制

### 核心原則

現金出資僅限用於**全體產品開發的共用資源**，個人專用設備或工具不計入股權計算。

### 轉換公式

**參數定義：**
```
POINT_TO_USD = $2 USD    # 現金轉點數匯率
```

**計算邏輯：**
```
1 點 = POINT_TO_USD = 0.1 小時勞務貢獻
```

**推導依據：**
- 參考月薪：100,000 TWD ≈ $3,333 USD（匯率 1:30）
- 月工時：176 小時（22天 × 8小時）
- 時薪：$19 USD/hour
- 1 點 = 0.1 小時 × $19 ≈ **$2 USD**

### 可計入項目 - 共用開發資源

| 項目類別 | 說明 | 範例 | 計算方式 |
| :--- | :--- | :--- | :--- |
| **雲端服務** | 產品運行所需的雲端基礎設施 | AWS、GCP、Firebase、Vercel | 年費總額 ÷ POINT_TO_USD = 點數 |
| **開發工具訂閱** | 團隊共用的開發工具 | GitHub Enterprise、Figma Team | 年費總額 ÷ POINT_TO_USD = 點數 |
| **域名與SSL** | 產品所需的網域與憑證 | Domain、SSL Certificate | 年費總額 ÷ POINT_TO_USD = 點數 |
| **API服務費** | 產品功能所需的第三方API | OpenAI API、地圖API、支付API | 實際使用費用 ÷ POINT_TO_USD = 點數 |
| **測試環境** | 專門用於測試的雲端環境 | Staging Server、Test DB | 年費總額 ÷ POINT_TO_USD = 點數 |

### 不可計入項目 - 個人資源

| 項目類別 | 說明 | 原因 |
| :--- | :--- | :--- |
| **個人開發設備** | 筆電、桌機、顯示器、鍵盤滑鼠 | 個人資產，非共用資源 |
| **個人測試設備** | 測試用手機、平板 | 個人持有，可移作他用 |
| **個人軟體授權** | 個人 IDE、個人 Adobe 訂閱 | 僅個人使用 |
| **辦公設備** | 桌椅、燈具、文具 | 個人使用，非專案專屬 |

### 計算範例

```
範例 1：雲端服務
- AWS 年費 $1,200
- 轉換點數：$1,200 ÷ POINT_TO_USD = $1,200 ÷ $2 = 600 點
- 計入維運價值 600 點/年 = 50 點/月

範例 2：API 服務
- OpenAI API 使用費 $500/年
- 轉換點數：$500 ÷ POINT_TO_USD = $500 ÷ $2 = 250 點
- 計入維運價值 250 點/年 = 20.8 點/月

範例 3：開發工具
- GitHub Team $48/年
- 轉換點數：$48 ÷ POINT_TO_USD = $48 ÷ $2 = 24 點
- 計入施工價值（一次性投入）
```

### 記帳與審核

- **記帳:** 所有現金支出需附發票或收據，記錄於專案財務表
- **審核:** 每季 review 支出是否符合共用開發資源定義
- **爭議處理:** 灰色地帶項目由全體貢獻者投票決定是否計入

> **註:** 若未來需調整 POINT_TO_USD 匯率，僅需修改參數定義即可，所有計算公式自動適用新匯率。
