## 產品利潤中心分配機制, Profit Center Attribution

- **核心概念:**
    - 為了讓貢獻者能直接從其負責的產品成功中獲益，我們採用 **「產品利潤中心」** 制度。
    - 每個產品組件視為獨立的利潤中心，其產生的營收 (Revenue) 扣除直接成本後，將優先分配給該組件的價值點數持有者。
- **分配邏輯:**
    1. **歸屬認定:** 營收需明確歸屬至特定產品組件 (e.g. Web Console 訂閱費, AI Advisor 加值服務費)。
    2. **分潤池:** 該產品的淨利潤形成該產品的 **「分潤池 (Dividend Pool)」**。
    3. **分配權重:** 個人在該產品的分潤權重 = (個人在該產品持有的價值點數) / (該產品總價值點數)。
    4. **範例:** 若 Web Console 產生 $100 淨利，而某 Frontend RD 持有 Web Console 10% 的點數，則他可分得 $10。
- **適用範圍:**
    - 此機制適用於 **Web 複雜報表** 與 **AI 個人化建議** 等具備獨立變現能力的加值產品。
    - 基礎產品 (記帳 App) 若為免費或低價引流，則其價值主要體現在推動整體生態系成長，可透過公司整體股權分紅彌補。

## 價值/定價一致性分析

| Tier | 月費 | 年費 | 累積施工 (點) | 年化維運 (點) | 維運/定價比 |
| :--- | ---: | ---: | ---: | ---: | ---: |
| Tier 0 (Local) | $0 | $0 | 20,250 | - | - |
| Tier 1 (Cloud) | $30 | $360 | 20,250 | 12,972 | 36:1 |
| Tier 2 (Management) | $90 | $900 | 31,340 | 22,512 | 25:1 |
| Tier 3 (Intelligence) | $140 | $1,400 | 42,840 | 36,912 | 26:1 |
| Tier B (Enterprise) | $500+ | $6,000+ | 52,410 | 57,456 | 10:1 |

**解讀:**
- Tier B 的維運/定價比最低 (10:1)，反映 B2B 服務的高維運成本
- Tier 1-3 的比值在 25-36:1，相對一致
- 這個比例表示：每 $1 定價，需投入 10-36 點的年化維運價值
