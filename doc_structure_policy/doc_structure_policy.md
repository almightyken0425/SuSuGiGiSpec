# 產品文件管理

## 產品規格 Product Specs

### Concepts

- **目的:** 闡述 App 的高層級設計 原則 與 理由。
- **內容規則:**
    - **應包含:** App 的核心設計原則, 闡述為何採用特定架構。
    - **應包含:** 為何本地優先、為何依時區計算、為何同步模型如此設計等 理由。
    - **禁止:** 包含具體的 UI 互動或 Logic 實作步驟。

### Views

- **目的:** 定義所有使用者可見的 畫面 與 互動。
- **內容規則:**
    - **應包含:** 所有畫面的 UI 佈局、元件結構、互動邏輯、與導航流程。
    - **禁止:** 描述業務計算邏輯, 例如報表如何加總。
    - **禁止:** 描述資料庫儲存邏輯, 例如存入哪個欄位。

### Controllers

- **目的:** 定義所有 背景運作 的業務邏輯, 這些邏輯不直接依賴 UI。
- **內容規則:**
    - **應包含:** 所有非 UI 的流程, 例如 App 啟動、報表計算、同步邏輯、資料轉換。
    - **禁止:** 描述 UI 佈局, 例如按鈕的顏色或位置。
    - **禁止:** 描述資料庫 Schema, 例如 id 是何種型別。

### Models

- **目的:** 定義 App 唯一的資料事實來源 Single Source of Truth。
- **內容規則:**
    - **應包含:** 所有資料庫的 Schema 定義、欄位、型別、索引。
    - **應包含:** 所有靜態 JSON 定義檔的結構。
    - **禁止:** 描述業務邏輯如何 使用 這些資料。
    - **禁止:** 描述 UI 互動如何 觸發 這些資料。

## 專案管理 Project Management

- **備註:** 專案的政策、規劃與待辦事項, 不屬於產品規格主體。

### Policy

- **目的:** 定義團隊協作的 規範。
- **內容規則:**
    - **應包含:** 所有與 如何撰寫 和 如何協作 相關的規範。
    - **應包含:** 規格撰寫風格、Git 流程、程式碼風格。

### Planning

- **目的:** 定義 開發 的藍圖與範疇。
- **內容規則:**
    - **應包含:** MVP 範圍、開發階段性藍圖、或檔案結構規劃。
